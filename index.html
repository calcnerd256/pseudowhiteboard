Pseudowhiteboard

A browser-based ZUI for collaborative, multiscale drawing.

browser-based

server

plugin-based architecture

I plan to evolve the design into a framework with plug-ins.

chat-ops as a database

Communication between clients looks like like an IRC chatroom.

client

Initial support is planned for two kinds of rich clients: JavaScript-heavy browser interfaces running on desktops and phones.

desktop

Connect a laptop or desktop computer to a projector or large TV and open Pseudowhiteboard in your Web browser, so everyone can see the big picture.

The desktop version of the interface shows the URL where the mobile users can find the mobile version of the interface.

mobile

Each meeting participant connects to the meeting with the Web browser on his or her smartphone.

Web

Pseudowhiteboard isn't a particularly good citizen of the World Wide Web. Unfortunately, while this page currently works without JS, the main application eventually won't — at least, the MVP won't.

ZUI

zooming

Mobile users will be able to pinch-zoom and pan around an unbounded canvas.

The desktop interface will zoom out to contain the entire drawing.

Mobile users can bookmark viewports to return to later. They can also summon each other or the desktop to bookmarked viewports.

collaborative

Multiple meeting participants can add content to the scene at the same time. Everyone can see everything everyone draws.

drawing

Mobile users can create content by adding strokes to the drawing.

multiscale

Strokes are rendered with a thickness and opacity appropriate for the zoom level of the viewport in which they were drawn.
